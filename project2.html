E-commerce Business Insights

Transforming raw e-commerce data into actionable insights with Python, Power BI, and interactive visualizations

Project Summary

Who Is This Project For?
This analysis is designed to give business analysts, e-commerce managers, marketing managers, or financial analysts a well-designed dashboard that can provide a deep level of understanding of an E-commerce business.

What Are the Goals?
The goal of this project is to transform raw sales data into meaningful business insights using Python for data cleaning and Power BI for visualization. This interactive dashboard helps answer critical business questions, including:
🎯 How does sales performance compare across categories
🎯 Which products are the top and worst sellers?
🎯 How does customer churn evolve over time?
🎯 How long does it take to fulfill an order?
🎯 Who are the best-performing suppliers?
🎯 What percentage of sales come from new customers?

What Will It Consist Of?
✔ ETL Process with Python – Extract, transform, and clean data.
✔ Data Modeling in Power BI – Build relationships between tables for optimized performance.
✔ Advanced DAX Calculations – Create KPIs for sales, order fulfillment, and profitability.
✔ Interactive Power BI Dashboard – Visualize insights with filters, slicers, and trend analysis.
✔ Publish Dashboard Online – Accessible via an online Power BI link.

What Tools Were Used?
🔹 Python (Pandas, NumPy) → Data cleaning & transformation.
🔹 DAX → Calculated measures for insights.
🔹 Power BI → Data modeling, visualization, and interactive dashboard creation.

Dataset Description

The dataset consists of 8 .CSV files representing different business operations:
Customers – Customer profiles with contact details.
Orders – Sales transactions, including order dates and total amounts.
Order Items – Individual products purchased within each order.
Payments – Payment methods and transaction statuses.
Products – Product catalog, including pricing and supplier details.
Reviews – Customer ratings and feedback on purchases.
Shipments – Shipment tracking details, including delivery times.
Suppliers – Supplier information for product sourcing.
The dataset required cleaning before analysis, ensuring accurate insights.

Analysis Steps

1️⃣ Defining the Business Questions

The first step was to explore the data from a business perspective, identifying valuable insights that could drive decision-making. By examining sales, customer behavior, supplier performance, and delivery trends, I determined how these datasets connect and what patterns could be uncovered. This ensured the analysis provided actionable insights rather than just beautiful sets of visual with low added value.

2️⃣ Data Cleaning & Transformation (ETL with Python)

Before analysis, the raw dataset was processed using Python (Pandas, NumPy):
✔ Checked for inconsistencies in data types (dates stored as text, missing numeric values).
✔ Standardized column names for consistency across datasets.
✔ Handled null values by filling missing data appropriately.
✔ Removed duplicate records for better accuracy.
✔ Fixed inconsistent data patterns (e.g., phone numbers, supplier codes).
You can find the Python code I used for data cleaning in My GitHub profile by clicking HERE.


3️⃣ Data Modeling in Power BI

The data is structured using a Snowflake Schema, with Orders as the central fact table, linking to Order Items, Payments, and Shipments for transaction details.
Dimension tables provide business context:
Customers connect to Orders and Reviews, linking purchases to feedback.
Products relate to Order Items, Reviews, and Suppliers, enabling analysis of sales, supplier performance, and ratings.
Shipments and Payments are linked to Orders for fulfillment and transaction tracking.
This normalized structure optimizes performance while allowing for detailed trend analysis and flexible filtering across multiple business dimensions.

4️⃣ Adding Key Calculated Measures (DAX)

Advanced DAX formulas were created to track performance:
✔ Promotion Expenses → (Price - Price at Purchase) * Quantity
✔ Order Fulfillment Time → (Order Date - Shipment Date)
✔ Delta Sales → Sales growth compared to the previous period.
✔ MTD (Month-to-Date) Sales → Tracks sales progress within the current month.
✔ YTD (Year-to-Date) Sales → Aggregates total sales for the current year.
✔ Last 30 Days Sales → Provides recent sales trends.
✔ AOV (Average Order Value) → Adjusts dynamically based on slicer selections.
✔ Order Frequency Trend → Measures how frequently customers make purchases.
✔ Sales & Sales Evolution Per Category → Compares product categories over time.
✔ Delivery Average Time → Calculates the average time taken to deliver orders.
✔ Successful Payment Ratio → Measures the proportion of successfully completed transactions.
✔ Average Shipment Days → Tracks the efficiency of shipments.
✔ Suppliers Sales Share → Determines each supplier's contribution to overall revenue.
✔ New Customer Sales Ratio → Analyzes the share of revenue from first-time buyers.

5️⃣ Visualization & Dashboard Creation in Power BI

This interactive Power BI dashboard provides a comprehensive view of sales performance, customer behavior, and supplier efficiency, enabling data-driven decision-making.

📌 General Overview
Users can track total sales, customer churn, order fulfillment time, and supplier contributions in real-time.
📌 Sales Tab
The Sales Tab allows users to segment sales by new vs. returning customers, filter by date range, and select product categories to analyze trends.
📌 Deep Insights & Custom Analysis
Users can:
✔ Compare new vs. returning customer sales.
✔ Select a specific date range for analysis.
✔ Filter by product category for focused insights.
🔹 Example: A user can analyze electronics sales for the last six months of 2024 to identify best-selling products and top 10 suppliers in that segment.

6️⃣ Sharing the Power BI Dashboard

You can use the Power BI dashboard of this project here:

Sharing Power BI dashboards with a free license can be challenging, and it took me a week to find a working solution. Since Power BI Publish to Web is often restricted, I had to set up a Microsoft Business trial account with admin access to enable public sharing.

📌 Guide to Share a Power BI Dashboard for Free:
1- Create a Microsoft Business Account (to have Admin Access to Power BI account) Sign up for a trial Microsoft 365 Business account : Mirosoft account registration link
Even after the trial expires, you keep Power BI access (but the Microsoft 365 account deactivates).

2- Upload & Open the Dashboard in Power BI Service
Sign in to Power BI Service using the same Microsoft 365 account.
Load your Power BI report into the Power BI workspace.

3- Enable Public Sharing in Power BI Admin Settings
Go to Settings → Admin Portal.
In Export and Sharing Settings, enable Publish to Web.
Disable "Block Public Internet Access" to allow external sharing.

4- Generate a Public Link for the Dashboard
Open your report in Power BI Service.
Click File → Embed Report → Publish to Web.
Copy the public link and share it!
⚠ Important: This method creates a public link, meaning anyone with the link can access the dashboard. It is not suitable for sensitive or company data.
